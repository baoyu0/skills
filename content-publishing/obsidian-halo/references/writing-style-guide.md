# 写作风格指南 — 结论标题与证据层级

> 借鉴 CyberPPT 方法论。用于 AI 在 Phase 3 排版优化时参考。
> 不在每次发文中都走完整流程，只在深度长文/原创文章中使用。

## 结论标题测试

H2 章节标题应当是**可辩护的判断句**，而非标签。

| ❌ 弱标题（标签） | ✅ 强标题（判断） |
|-----------------|-----------------|
| `## 3. 市场概览` | `## 3. 市场增长正在修复，但价值正在向结构性优势赛道转移` |
| `## 5. 消费者分析` | `## 5. 核心消费者继续扩大，但购买逻辑更加分化` |

强标题必须是「可以被页面证据挑战并守住」的完整判断。如果标题改成判断句后觉得证据撑不住，说明章节内容本身需要补强，而不是退回标签。

每写完一个 H2，花 5 秒问自己：「这个标题有没有结论？」

## 证据层级结构

每节正文按以下顺序组织，不要只摆图表/数据不说明它证明了什么。一篇长文中至少 2-3 节完整走完这个层级：

1. **结论** — 该节的核心判断（对应 H2/H3 标题）
2. **主证据** — 最强的一两个数据、事实或引用来支撑结论，标记来源
3. **次级证据** — 细分数据、对比、趋势拆解（可选，看篇幅）
4. **解读** — 这些证据放在一起说明了什么（连接证据和结论的逻辑链）
5. **含义 / SO WHAT** — 读者为什么要关心这个？影响是什么？下一步行动？

### 最低要求

每节至少要有「结论 + 证据 + 含义」三层。避免「只有数据没有解读」或「只有观点没有证据」。

### 快速检查

```python
# 在 execute_code 中快速检查每节的证据完整性
with open(path) as f:
    text = f.read()

sections = text.split('## ')
for s in sections:
    if not s.strip():
        continue
    has_conclusion = True  # heading 本身就是结论
    has_evidence = any(kw in s for kw in ['例如', '数据', '根据', '研究表明', '统计'])
    has_meaning = any(kw in s for kw in ['这意味着', '所以', '因此', '建议', '下一步'])
    if not has_evidence or not has_meaning:
        print(f'⚠️ 章节 "{s[:50]}..." 缺少 {"证据" if not has_evidence else ""} {"含义" if not has_meaning else ""}')
```
