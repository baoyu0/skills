# Heading 层级修复准则

## 为什么需要

用户明确要求"至少 3 级标题结构"，反感全篇只有 H2 的扁平文章。**纯 H2 无 H3 = 扁平目录，禁止。** 即使叙事文章也需在章节内抽 1-2 个子节主题转为 H3，保证目录可嵌套。

## 文章类型与处理方式

### 类型 A：已有 markdown heading 的文章

如果文章自带 `##` / `###` 标记，直接用 `auto-number.py` 编号即可，很少需要 AI 额外修改。

### 类型 B：纯叙事文章（无标题标记）

X/Twitter 剪藏、访谈记录、消息聚合等通常只有纯文本段落 + 内嵌图片。**这是最需要 AI 手工重建 heading 的类型。**

重建原则：
1. **H1** = 文章标题（auto-number 自动从 frontmatter title 补）
2. **H2** = 按主题切分章节（通常 4-6 节）
3. **H3** = 每章至少拆 2 个，从正文中提取子主题

### 如何从叙事文本中拆出 H3

寻找以下自然子节信号：

| 信号 | 示例 | 做法 |
|------|------|------|
| **粗体短句独立成行** | `**PRD 没死。**` | 转为 `### N.1 PRD 没死` |
| **引言引用** | `**"不要砍掉产品角色"**` | 取核心短语转 H3, 去掉引号 |
| **转折/结论句** | `"问题不是文档和原型哪个好"` | 提炼为 `### N.1 选对媒介` |
| **图片自然分隔** | 图片前后主题明显切换 | 把图片后首段的一句话提炼为 H3 |
| **数字列举** | 内部有"第一/第二"或数字列表 | 每一项对应一个子节 |

### 叙事文章 H3 拆解示例

```
## 1. PRD 没死
  原正文：他说得很直："I actually don't believe this at all." ...
  → ### 1.1 文档 vs 原型的误区
  → ### 1.2 选对媒介

## 2. 不要砍掉产品角色
  原正文：他说已经有公司在这么干了。PM 全撤掉...
  → ### 2.1 PM 角色的价值
  → ### 2.2 边界变薄不等于角色消失

## 3. 设计流程没死透
  → ### 3.1 流程已死的真相
  → ### 3.2 信号与媒介脱钩
```

**每章 2-3 个 H3 足够**，不要硬凑。如果一章只有 2-3 段话，可以只加 1 个 H3 甚至不加（但 H2+无H3 = 扁平，要避免）。

### 类型 C：X Article blockquote 标题

X（Twitter）的文章常用 **blockquote 格式做标题**，不是 markdown heading：

```markdown
> **一、章节名**    # 转为 ## N. 章节名
> 二、章节名        # 转为 ## N. 章节名（无加粗）
> 1. 章节名         # 转为 ## N. 章节名（阿拉伯数字）
```

转换要点：
- 去掉 `> ` 前缀
- 中文数字（一二三）改为阿拉伯数字（1. 2. 3.）
- 去掉多余加粗
- 文件其他地方如有指向章节的引用（如 `> ① 前端审美 ② 架构规划 ③ 代码编写`），也一并更新编号

## 修复步骤

### 1. 分析当前结构

```bash
search_files pattern="^#{1,4} " path="<文件绝对路径>" output_mode="content"
```

### 2. 判断处理路径

| 当前结构 | 做法 |
|----------|------|
| 已有 `#` + `##` + `###` 三级 | 直接编号 + 排版，无需重建 |
| 纯文本无标题 | **强制 H1→H2→H3 三级**：首段 H1，按主题切 H2，每章至少 2 个 H3 |
| 只有 H2 无 H3 | **纯 H2 无 H3 = 扁平目录**，必须从正文中抽子节主题建 H3 |
| 只有 `##` + `###`（无 H1） | 正文首行补 `# 标题` |
| blockquote 标题 | 转为 `## N. 标题` 后，再检查是否需要拆 H3 |

### 3. 避免的误区

- ❌ **只建 H2 不拆 H3** → 扁平目录，必须禁止
- ❌ **把长段落整体设为 H3** → H3 应是子节标题，不是正文
- ❌ **标题中有编号重复** → `### 4.1 一、术语先收敛` 中的 `一、` 多余
- ❌ **H1 中加粗** → `# 关于 **PRD** 的反思` → 恢复为纯文本

## Heading 数量参考

| 文章字数 | 合理 H2 数 | 合理 H3 数 |
|----------|-----------|-----------|
| 2000-4000 | 3-5 | 4-8 |
| 4000-8000 | 4-6 | 6-12 |
| 8000+ | 5-8 | 10-20 |
