# 结构化 Prompt 排版方案

当文章以分享 AI 视频/代码提示词为主要内容时，采用两层排版结构。

## 判断标准

文章正文同时包含以下特征时触发此方案：

- 属性标记：`【风格】` / `【时长】` / `【场景】` / `【角色】` 等
- 场景时间码：`[00:00-00:03]` 分镜头标记
- 角色描述：Q 版动漫贴纸、扁平 2D 质感等角色设定

## 两层结构

### 第一层：元属性（【风格】/【时长】/【场景】/【角色】）

**格式**：正文加粗标头 `**【属性名】** 值`

这些是 prompt 的参数设定，属于说明性质。保持为普通正文段落，加粗标头让读者一眼识别参数名。

**示例：**
```markdown
**【风格】** 真人实拍+2D动漫贴纸合成搞笑短视频...
**【时长】** 10秒
**【场景】** 真实街边早餐摊第一人称视角...
**【角色】** Q版动漫贴纸角色...
```

### 第二层：分镜/场景层

**格式**：每个场景独立一个 ` ```text ` 代码块

**理由：**
- 每个代码块有独立复制按钮，读者可单独提取某场景
- 视觉上清晰分隔不同镜头，阅读节奏更好
- 不会因一个大代码块拖长页面

**示例：**
```markdown
```text
[00:00-00:03] 镜头1：酱料指挥失控（Sauce Command Opening）
第一人称俯视POV：真人右手正用刮板把面糊摊成圆形...
音效：铁板滋滋声 + 刮板摩擦声 + 酱料飞溅声 + ...
```

```text
[00:03-00:06] 镜头2：薄脆雪崩（Crispy Cracker Avalanche）
...
```
```

## 分隔

元属性与分镜之间用 `---` 分隔。分镜部分前加 H3：

```markdown
---

**【角色】** ...（元属性结束）

### 分镜详情

```text
[00:00-00:03] 镜头1...
```
```

## 多版本处理

同一篇文章包含多个 prompt 版本（不同角色/场景但同一框架）时：

- 每个版本之间用 `---` 分隔
- 版本标题用 `## N. 版本名`
- 每个版本内部独立按两层结构排列
- 结尾加 `## 总结` 收束两个版本

**示例：**
```markdown
---

## 2. 再来一版办公室捣蛋鬼

[视频]
---
**【风格】** ...
**【时长】** 10秒
**【场景】** ...
**【角色】** ...

```text
[00:00-00:03] 镜头1：键盘蹦迪
...

## 总结

两个版本风格一致，角色和场景不同...
```
```

## 与 X 剪藏 cleanup 的配合

如果文章来自 X 剪藏（含 `## 帖子` / `## 对话` 等结构）且内容是 prompt，必须先做 X 清理：

1. 剥离 `## 帖子`（保留或不保留看内容质量）
2. 剥离「引用」「发布你的回复」「由 AI 生成」等 X UI 元数据
3. 确认只有 prompt 核心内容后，再套用本方案
4. 元属性（【风格】等）已自然存在于正文中，直接保留为加粗标头
5. 场景描述按 `[00:XX-00:XX]` 时间码拆成独立代码块
