# 拼好码 · Code Assembly

> 能复用时不重造，能编排时不发明。

每次编码任务开始时加载本文件。方法论详解见 `SKILL.md`。

## 执行流程

```
需求 → 拆能力 → 搜①官方 → 搜②社区 → 搜③平台 → 评估 → 选组合 → 定边界 → 写胶水 → 验证
```

### 7 步速查

1. **写清需求** — 目标、输入、输出、约束、验收标准
2. **反向搜索** — ①官方 CLI/SDK/内置能力 → ②社区工具 (stars>500) → ③平台/SaaS
3. **评估候选** — 更新日、许可证、stars、文档、生产案例
4. **选组合** — 记录为什么选、为什么不选、为什么不自研
5. **定边界** — 适配层、接口签名、回滚路径
6. **写胶水** — 只写连接/编排/配置/规则/测试，不重写工具内部
7. **验证** — 类型检查、集成测试、验收逐条过、留证据

### 反模式

| ❌ 别这样 | ✅ 要这样 |
|-----------|----------|
| 看到需求直接写代码 | 先搜方案再动手 |
| 凭印象选工具 | 查 stars、更新日、许可证 |
| 先自研不行再换 | 先搜，评估完再写 |
| 自研一切 | 只写胶水代码 |

### 原则

- 搜索不可跳过
- 评估不可省略
- 记录不可偷懒
- 回滚路径必须有
- 验证才等于完成
