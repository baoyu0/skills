# 拼好码 · Code Assembly

> 能复用时不重造，能编排时不发明。

**拼好码**是一套 AI 编程方法论，由 **Hermes Agent** 在生产实战中提炼而成。它要求从「实现者心态」转向「整合者心态」：不是看到需求就让 AI 直接自研，而是先识别已有成熟能力、评估成熟度、设计适配边界，再用最少自研完成业务闭环。

成熟生态承担通用复杂度，胶水代码连接业务流程，自研只服务不可替代的业务差异。

---

## 思维转变

| 从（实现者） | 到（整合者） |
|-------------|-------------|
| "这个功能我写一下" | "这个功能有没有现成的 CLI/包/SDK？" |
| "先跑起来再说" | "先搜方案，评估完再动手" |
| "自己写更可控" | "用成熟工具，出问题时上游修" |
| "就一个小功能不用查" | "再小的功能也可能有成熟库" |
| "这个工具我熟悉" | "确认一下有没有更新的替代品" |

## 7 步工作流

```
需求 → 拆能力 → 搜①官方 → 搜②社区 → 搜③平台 → 评估 → 选组合 → 定边界 → 写胶水 → 验证
```

### 1. 写清需求
对齐目标、输入、输出、约束、验收标准。

### 2. 反向搜索
按优先级三层搜索：**① 官方能力** → **② 社区成熟工具** → **③ 平台服务**
每层不可跳过。如果当前层找不到，拆解能力领域到更细粒度继续搜。

### 3. 评估候选
逐项过评估矩阵：维护状态、许可证、文档质量、生产案例、生态兼容、替换风险、接入成本。**不满足标准的果断放弃。**

### 4. 选择组合
确定采用方案，**并记录决策理由**——"为什么不选 X"比"为什么选 Y"更有长期价值。

### 5. 设计边界
固定接口契约：输入输出模型、适配器包装、错误处理、依赖隔离、回滚路径。

### 6. 生成胶水代码
只写连接、适配、编排、配置、业务规则和测试。**绝不重新实现成熟能力本身的内部逻辑。**

### 7. 验证交付
类型检查、集成测试、验收标准逐条过、留下证据、记录回滚路径。

## 实战案例

### ✅ Obsidian → Halo 发布流程
找到 `@halo-dev/cli` 的 `import-markdown` 命令后，删除 200+ 行自研 API 胶水代码。发布走官方 CLI，本地只做 markdown 处理。

### ✅ BrowserAct 评估后放弃
30 分钟评估发现核心能力已被 obu 覆盖，stealth-extract 需付费 key。❌ 果断不装，省了 2 天试错。

### ✅ Karing GitHub 下载修复
根因是路由规则缺 CDN 域名。**零代码**，只改了两行配置。

### ❌ X Article 视频下载
确认无任何 CLI 工具支持后，选择务实的手动方案（猫抓扩展 + poster 兜底）。及时识别"无解场景"并降级，也是拼好码的一部分。

## 核心理念

| 原则 | 含义 |
|------|------|
| 搜索不可跳过 | 不论任务大小，先走完搜索链再动手 |
| 评估不可省略 | 每个候选必须过矩阵，看完 stars/更新日/许可证 |
| 记录不可偷懒 | 决策理由写下来，"不选 X"比"选 Y"更有价值 |
| 回滚路径必须有 | 没有回滚路径的方案不是可交付方案 |
| 验证才等于完成 | 验收标准逐条过完才叫"做完" |
